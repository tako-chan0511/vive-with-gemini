## 1. 自宅VPNと閉域網接続の完全ガイド（表紙）
![VPN1.自宅VPNと閉域網接続の完全ガイド](/images/network/VPN1.自宅VPNと閉域網接続の完全ガイド.jpg)

### 図の要旨
- リモートワークでは「社内ネットワークの外側（自宅・移動先）」が業務の主戦場になり、**見えない脅威**（盗聴・なりすまし・設定ミス・端末汚染など）と、**切り分けが難しい障害**（VPN/閉域網/DNS/プロキシ/端末要因）が増えます。
- 本資料は「なぜ危ないのか」「どこで詰まるのか」を、セキュリティとトラブルシューティングの両面から整理します。

### 詳細解説（用語）
- **VPN**：端末〜社内/クラウドの間に「暗号化トンネル」を作る仕組み。盗聴対策には強いが、設定次第で“抜け道”も作れる。
- **閉域網**：インターネットを経由しない（または論理的に分離された）ネットワーク。安全性が高い一方、経路やDNSなどが特殊で、切り分けが難しくなりがち。
- **見えない脅威**：ユーザーには“普通に繋がっているように見える”が、実は盗聴/中継/改ざんされている、あるいは端末が踏み台化している状態。

### Tips（実務）
- 最初に「現場で使う切り分け手順（チェックリスト）」を決め、障害時に同じ観点でログを取ると再発防止が早いです（後半のコマンド章につながります）。

## 2. 防御の境界線は「社内」から「自宅」へシフトした
![VPN2.防御の境界線は「社内」から「自宅」へシフトした](/images/network/VPN2.防御の境界線は「社内」から「自宅」へシフトした.jpg)

### 図の要旨
- 以前：社内LANの“入口”にFW/IDS/Proxyなどを置き、境界（Perimeter）で守る。
- 現在：自宅回線・自宅ルーター・個人端末・公衆Wi-Fiが境界になり、**境界が分散**した。

### 詳細解説（用語）
- **境界防御（Perimeter Security）**：ネットワークの出入口で守る考え方。社内に人と端末が集まる前提で強い。
- **分散した境界**：在宅・出張・サテライトオフィスで“入口が増える”状態。入口が増えるほど、設定ミスや弱点も増える。

### Tips（実務）
- 「会社が守ってくれる」前提を一度捨て、最低限 **(1)自宅ネットワーク (2)端末 (3)接続先** を自分で点検できるようにするのが最短です。

## 3. 接続方式の違いを知る：インターネットVPN vs 閉域網
![VPN3.接続方式の違いを知る：インターネットVPNと閉域網](/images/network/VPN3.接続方式の違いを知る：インターネットVPNと閉域網.jpg)

### 図の要旨
- **インターネットVPN**：経路はインターネット。暗号化で守る。便利だが“ネットの状態”に影響される。
- **閉域網**：経路が分離/制御される。安定・安全になりやすいが、構成が特殊で“繋がらない時の原因”が増える。

### 詳細解説（用語）
- **暗号化**：通信内容を第三者が読めない形にすること。盗聴には強いが、端末が乗っ取られていたら“正規の暗号通信”として盗まれます。
- **経路（ルーティング）**：どの道で目的地へ行くか。閉域網はこの“道”がインターネットと違うため、DNSやプロキシの設定も変わることがあります。

### Tips（実務）
- 「VPNは繋がったのに社内システムに入れない」は、**ルート/DNS/プロキシ**のどれかが原因のことが多いです（後半の切り分けコマンドへ）。

## 4. 「暗号化」は「無敵」ではない：スプリットトンネリングのリスク
![VPN4.「暗号化」は「無敵」ではない：スプリットトンネリングのリスク](/images/network/VPN4.「暗号化」は「無敵」ではない：スプリットトンネリングのリスク.jpg)

### 図の要旨
- **フルトンネル**：全通信がVPNを通る。監視/制御しやすいが遅くなることも。
- **スプリットトンネル**：社内宛だけVPN、その他は直インターネット。速いが、**危険な通信が混ざりやすい**。

### 詳細解説（用語）
- **スプリットトンネリング**：通信先で“VPNを通す/通さない”を分ける設計。
- リスク例：
  - 端末がマルウェア感染→社内向け通信も同じ端末から行われる（暗号化していても“正規の端末”が悪い）。
  - DNSが社外側に向く→社内名解決ができず「繋がらない」に見える／偽サイト誘導リスク。

### Tips（実務）
- セキュリティ優先なら基本はフルトンネル。ただし業務要件でスプリットが必要な場合は、**端末防御（EDR/AV）とDNS統制**をセットで考えるのが現実的です。

## 5. 第1の防衛線：自宅ルーターの「踏み台」化を防ぐ
![VPN5.第一の防衛線：自宅ルータの「踏み台」化を防ぐ](/images/network/VPN5.第一の防衛線：自宅ルータの「踏み台」化を防ぐ.jpg)

### 図の要旨
- 自宅ルーターが侵害されると、通信の盗聴・改ざん・偽サイト誘導などの“入口”になります。

### 詳細解説（初心者向け）
- **踏み台（Bot/踏み台化）**：攻撃者が機器を乗っ取り、別の攻撃や盗聴の中継点にすること。
- まず押さえるべき設定（優先度順）：
  1) **管理画面パスワード変更**（初期ID/初期PWは最優先で変更）  
  2) **ファームウェア更新**（脆弱性修正）  
  3) **WPA2/WPA3の強固なWi-Fi鍵**（推測されにくい長いパスフレーズ）  
  4) **リモート管理（WAN側からの管理）を無効化**  
  5) **UPnP無効化**（不要なら）  
  6) **ゲストWi-Fi**でIoT/来客端末を分離

### Tips（実務）
- ルーターが業務影響の原因になる典型例：  
  - DNS設定が勝手に書き換わる（怪しいDNSへ）→社内サイトが引けない/偽サイトへ  
  - ルーター負荷過多→VPNが頻繁に切れる（ログが残りにくい）  
- 可能なら「仕事用Wi-Fi（SSID）」を分け、家族端末/IoTを同居させないと安定します。

## 6. 第2の防衛線：物理セキュリティとデバイス衛生
![VPN6.第二の防衛線：物理セキュリティとデバイス衛生](/images/network/VPN6.第二の防衛線：物理セキュリティとデバイス衛生.jpg)

### 図の要旨
- ネットワークが安全でも、端末そのものが覗かれたり持ち去られたりすると終わります。
- “端末を清潔に保つ（衛生）”が重要。

### 詳細解説（初心者向け）
- **物理セキュリティ**：画面の覗き見、置き忘れ、家族の誤操作など“現実世界”のリスク。
- **デバイス衛生**：OS/ブラウザ更新、不要ソフトの削除、ウイルス対策、ローカル管理者権限の扱いなど。

### すぐ使える操作例
- 画面ロック：  
  - Windows：`Win + L`  
  - macOS：`Control + Command + Q`
- 自動ロック（放置対策）：5〜10分程度を推奨（組織ポリシーに従う）

### Tips（実務）
- “家族が触れるPC”は事故が起きやすいので、可能なら業務専用端末に寄せるのが一番コストが安いです。

## 7. 公衆Wi-Fiの罠：無料は「タダ」ではない
![VPN7.第三の防衛線：講習Wi-Fiの落とし穴](/images/network/VPN7.第三の防衛線：講習Wi-Fiの落とし穴.jpg)

### 図の要旨
- 公衆Wi-Fiでは、悪意ある中継（盗聴・改ざん）や偽アクセスポイント（Evil Twin）が起き得ます。

### 詳細解説（用語）
- **Evil Twin**：正規のSSIDに似せた偽Wi-Fiを用意し、接続させて通信を覗く手口。
- **MITM（Man-in-the-Middle）**：通信の途中に割り込み、盗聴/改ざんする攻撃。

### 対策（現場で効く順）
1) 可能なら **テザリング**（最優先）  
2) 公衆Wi-Fiを使うなら **VPN必須**  
3) Windowsのネットワーク種別を **「パブリック」**（ファイル共有オフ）  
4) 自動接続を切る（「以前接続したWi-Fiに勝手につながる」を防ぐ）

### Tips（実務）
- 「HTTPSだから安全」は半分だけ正しいです。HTTPSでも、偽の証明書を受け入れてしまう/端末が汚染されていると破綻します。

## 8. トラブルシューティング入門：感覚ではなくデータで特定する
![VPN8.接続トラブルシューティング：自己診断のツールキッド](/images/network/VPN8.接続トラブルシューティング：自己診断のツールキッド.jpg)

### 図の要旨
- “繋がらない”は原因が複数あり、**順番に潰す**のが最短。
- OS標準コマンドで「端末→ルーター→回線→経路→DNS→相手」のどこが壊れているか切り分けます。

### 詳細解説（考え方）
- 原則：**近いところから遠いところへ**  
  1) 自分のIP/DNS/ゲートウェイ（端末の状態）  
  2) ルーターまで届くか  
  3) インターネット（または閉域網）の外へ出られるか  
  4) 目的地までの経路  
  5) DNS（名前解決）  
  6) ポート/アプリ（TCP接続）

### Tips（実務）
- “現象メモ”を残すだけでも、再現性が上がり、ヘルプデスク/ネットワーク担当に渡す情報が揃います（後述の各コマンドの見どころをメモ化）。

## 9. Step1：IP設定を確認する（ipconfig）
![VPN9.STEP1：自身の状態を知る（ipconfig）](/images/network/VPN9.STEP1：自身の状態を知る（ipconfig）.jpg)

### 図の要旨
- 端末が「どのIPを持ち、どのDNSを使い、どこを出口（Gateway）にしているか」を見る。

### コマンド解説（初心者向け）
```bat
ipconfig /all
````

見るべきポイント（重要順）：

* **IPv4 Address**：例 `192.168.x.x` が一般的。

  * `169.254.x.x` なら **DHCP失敗（IPを貰えてない）** の疑いが強い。
* **Default Gateway**：だいたいルーターのIP。これが空/変だと外へ出られない。
* **DNS Servers**：社内名解決が必要な環境では、ここが指定通りでないと「社内サイトだけ開けない」になりやすい。

関連コマンド：

```bat
ipconfig /flushdns
```

* DNSキャッシュを消して、古い名前解決結果が残る事故を防ぎます。

### Tips（実務）

* VPN接続後に `ipconfig /all` を2回（接続前/後）取ると、**DNSやルートがVPNでどう変わったか**が見えて、切り分けが一気に楽になります。

## 10. Step2：疎通確認（ping）

![VPN10.STEP2：到達性を確認する（ping）](/images/network/VPN10.STEP2：到達性を確認する（ping）.jpg)

### 図の要旨

* 相手まで“届くか”と“遅延/ロス”をざっくり測る。

### コマンド解説（初心者向け）

```bat
ping 8.8.8.8
ping example.com
```

* 数値の見方：

  * **time=xxms**：遅延（大きいほど遅い）
  * **Request timed out**：応答なし（ただし“相手がpingを拒否している”可能性もある）

### 注意（閉域網/社内NWあるある）

* セキュリティ設定で **ICMP（ping）を遮断**していることがあります。
  → pingが通らない＝必ずしも死んでいる、ではない。
  その場合は後述の **tracert / nslookup / ポート確認**へ。

### Tips（実務）

* まずは「ルーター」に打つと切り分けが早いです（例：ゲートウェイが `192.168.0.1` なら `ping 192.168.0.1`）。

## 11. Step3：経路のどこで止まるか（tracert）

![VPN11.STEP3：経路のボトルネックを探す（tracert）](/images/network/VPN11.STEP3：経路のボトルネックを探す（tracert）.jpg)

### 図の要旨

* 目的地までの“道のり（ホップ）”を表示し、どこで詰まっているかを見る。

### コマンド解説（初心者向け）

```bat
tracert example.com
```

* **hop（ホップ）**：中継点。数字が増えるほど遠くへ行っている。
* `* * *`：応答が返ってこない中継点（遮断/混雑/設定による）。

### 読み方のコツ

* 早い段階（1〜2hop）で止まる：自宅ルーター/回線/PC側の問題が濃厚
* 中盤で止まる：ISP/インターネット経路の問題の可能性
* VPN配下の宛先で止まる：VPNゲートウェイ以降（社内側）の制御/障害の可能性

### Tips（実務）

* “どこから先がVPNの世界か”が分かると強いです。VPN接続後に `tracert` を取り、**VPNゲートウェイらしきホップ**をメモすると原因説明がしやすくなります。

## 12. Step4：名前解決と通信状態（nslookup / netstat）

![VPN12.STEP4：名前解決と接続状況の詳細](/images/network/VPN12.STEP4：名前解決と接続状況の詳細.jpg)

### 図の要旨

* 「IPには届くが、名前（URL）だと開かない」ならDNSの疑い。
* 「アプリが本当に接続しているか」を見るならnetstat。

### コマンド解説（初心者向け）

DNS確認：

```bat
nslookup intranet.example.local
nslookup intranet.example.local 10.0.0.53
```

* 1行目：PCが設定しているDNSで引く
* 2行目：DNSサーバを指定して引く（“DNSが間違っている”切り分けに強い）

通信状態：

```bat
netstat -ano
netstat -ano | findstr :443
```

* `-a`：全接続表示、`-n`：IP/ポートを数値で表示、`-o`：PID（プロセスID）表示
* `findstr` はWindowsの文字列フィルタ（Linuxの `grep` 相当）

PIDからプロセス特定（必要に応じて）：

```bat
tasklist | findstr 1234
```

### Tips（実務）

* 「社内サイトだけ開かない」は、DNSが“社外向け”のままになっているケースが多いです。VPN接続時にDNSが切り替わる設計か、固定DNSが必要かを確認します。

## 13. VPNの先へ：ゼロトラスト・セキュリティ

![VPN13.VPNの先へ：ゼロトラストセキュリティ](/images/network/VPN13.VPNの先へ：ゼロトラストセキュリティ.jpg)

### 図の要旨

* “ネットワークの内側＝安全”という前提が崩れたため、**常に検証する**ゼロトラストが重要になる。

### 詳細解説（用語）

* **ゼロトラスト（Zero Trust）**：

  * *Trust Nothing, Verify Everything*（何も信頼せず、常に検証）
* 代表要素：

  * **MFA**：多要素認証（パスワード漏洩に強い）
  * **端末準拠（Device Posture）**：OS更新/暗号化/EDRなど条件を満たす端末だけ許可
  * **最小権限**：必要最低限だけアクセスさせる（横展開を防ぐ）

### Tips（実務）

* “VPNを強くする”だけでは限界があります。ID（認証）と端末（健全性）を組み合わせると、運用で事故が減ります。

## 14. まとめ：自分で守るためのチェックリスト

![VPN14.セキュリティ＆トラブルシューティングチェックリスト](/images/network/VPN14.セキュリティ＆トラブルシューティングチェックリスト.jpg)

### 図の要旨

* リモート環境は「環境」「デバイス」「接続」「行動」の4点セットで守る。

### チェック項目（例）

* 環境（自宅ネットワーク）

  * ルーター管理PW変更 / FW更新 / リモート管理OFF / UPnP見直し
* デバイス

  * OS/ブラウザ更新、ウイルス対策、画面ロック、ディスク暗号化（BitLocker等）
* 接続

  * 公衆Wi-Fiは原則避ける、使うならVPN、ネットワーク種別はPublic
* 行動

  * 不審リンク/添付を開かない、業務用と私用の分離、困ったらログを取って相談

### Tips（実務）

* “完璧にやる”より「事故が起きやすい上位3つ（ルーター/端末更新/公衆Wi-Fi）」から潰す方が効果が出ます。

## 15. Appendix：コマンド早見表（Windows中心）

![VPN15.付録：コマンドラインのクイックリファレンス](/images/network/VPN15.付録：コマンドラインのクイックリファレンス.jpg)

### 図の要旨

* 障害切り分けの“最低限セット”を一覧化。

### コマンドと用途（初心者向けの補足）

* IP/DNS/ゲートウェイ確認：

```bat
ipconfig /all
```

* DNSキャッシュクリア：

```bat
ipconfig /flushdns
```

* 疎通（遅延・ロスの目安）：

```bat
ping <IP or host>
```

* 経路（どこで止まるか）：

```bat
tracert <host>
```

* DNS（名前解決の正誤）：

```bat
nslookup <host>
```

* 通信状態（接続・待受・PID）：

```bat
netstat -ano
```

### Tips（実務）

* ヘルプデスク/ネットワーク担当に渡すと喜ばれる最小ログ：

  * `ipconfig /all`（接続前/後）
  * `nslookup 対象ホスト`
  * `tracert 対象ホスト`
  * 可能なら「発生時刻」「どの回線（自宅/テザリング）」「VPN接続有無」も添える
